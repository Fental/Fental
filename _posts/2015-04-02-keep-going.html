坚定信念，不再迷茫：《前端这些年》及Ovilia在知乎回答的读后感

感悟与反省

Ovilia在她的网站进击之路上，也不是一开始就很坚定，目标明确。一路走下来，也并不是只是接触前端的东西，学的东西也是很杂，并根据项目需求学习新知识并不断应用实践。PHP呀，python之类的她也接触。

前端这几年，她在这个领域找到了兴趣，在实践中磨练技术，不断学习新的技术，并得意于她那时候的设计和成果。（大家其实都一样，刚做出点小东西就会觉得特别自豪。尽管回头看看会觉得很羞耻…）

一步一个脚印，Ovilia也并非一开始就会飞。

而我呢？接触前端一年，啃着几本教材，对着上面的基础知识反复看来看去，没有应用于实践（虽说啃基本知识，但也没有记得多少，毕竟没有在实践中应用，难以加深印象。）；一直在踌躇是不是要继续学习JQuery，什么时候开始学习Bootstrap, Less, AMD, CMD…

我跟她的区别：

  1. 反正智商和学习能力上肯定是有一些差距的，毕竟一个华工，一个交大。
  2. 太贪婪了，不懂如何剪枝。我总是想得太多，做得太少，将时间浪费在所谓的“思考”上。

    我们都知道蛮力通常是程序员最后的选择，要学会剪枝才能事半功倍。

  3. 脑洞虽大，但是没有将自己的灵感和创意付诸实践

    其次，就是我脑洞比较大，经常有些 idea，所以就会去做出来。
    比如学易经变卦的时候，我想知道每一爻的变化会让卦象从什么变成什么，所以做了 变卦·易经64卦 ；看到 Gmail 附件折角的效果，想用 CSS 实现，做成一个 jQuery 的插件，所以有了 jCorner ；希望在豆瓣网站上看到可供下载的 ReadFree 上的电子书，所以做了 Chrome 插件 豆瓣 ReadFree 传送门 ；经常在网页里看国学的书想查汉典，所以做了Chrome 插件 汉典划词查询 ；等等……

    相反，如果你本身并不是一个有很多想法的人，你就会觉得有这么多创意其实要比实现更困难，最后只能是自我否定地觉得为什么别人这么厉害，我什么都没想法。其实不是谁厉害，只是每个人的性格不同所以需要采用不同的激励方法。了解你自己，用你的办法激励自己，让自己变得更优秀。

  4. 没有学习计划以及对未来的规划，Ovilia会列出自己的学习列表，有明确的学习目标。

    以下是Ovilia 2014的学习列表：

    • Bootstrap
    • AngularJS
    • RequireJS
    • Node.js
    • JavaScript Template Engine
    • JavaScript Language Feature
    • JavaScript Design Pattern
    • LESS
    • SASS

    Ovilia 对学习内容的看法：

    我年初定的2014学习列表里的九项，如今才学了Bootstrap、LESS两项而已……接下来最可能要接触到的就是Angular.js和Node.js了；JavaScript语言特性的话，很难说什么时候是学完了，但至少经典的几本书最近都没怎么看；JavaScript模板引擎我觉得不用学了，Angular.js应该可以代替它；Require.js应该不怎么需要学，但也没实际项目需要用到，所以还没操练过；其它几项倒也不是很要紧。

    所以，总结来看，接下来就是把重点放在Angular.js和Node.js上，并且继续学习JavaScript。

改变与规划

  1. 学会放弃，不要整天纠结说到底是要干前端还是后台。已经在前端上培养了兴趣，前端，就是干！当然，虽然做不到全才，样样精，但是后台的东西还是需要了解并掌握一些。
  2. 发挥脑洞！多记录自己的灵感，并尝试生产出来。

  3. 指定学习列表，明确自己接下来的学习内容：

    • JQuery
    • Bootstrap
    • RequireJS || Angular.js
    • LESS
    • SASS
    • 阅读JS一些经典书籍

    其中JQuery是当前比较流行的类库，学习时不仅要学学怎么用，还要看看JS源码。

    模块化在前端开发中也是非常重要，RequireJS || Angular.js！！！

    学会使用Bootstrap框架搭建网站，学会使用LESS || SASS编写CSS

    经典书籍包括：红宝书，犀牛书、蝴蝶书、《WebKit技术内幕》 等等
